For each of the following workloads, the deliverable is a short written analysis (~100–150 words each) that:

1. Identifies whether the workload is a good fit for GPU acceleration, and why.
2. Names the most likely performance bottleneck if implemented on a GPU (compute-bound, memory-bound, divergence-bound, or transfer-bound).
3. States which of CPU / GPU / FPGA / ASIC would be the most appropriate platform overall, given everything studied this term, and why.

**Workload 1:** Applying a brightness/contrast filter to every pixel of a 4K video stream in real time.

**Workload 2:** Running a chatbot's neural network to generate one response to a single user query, where the model must reply within 200ms.

**Workload 3:** A high-frequency trading system that must react to a single incoming market data packet within 200 nanoseconds.

**Workload 4:** Training a large image-classification neural network on a dataset of 10 million images, where training time (hours to days) is the primary cost driver.

**Workload 5:** A recursive pathfinding algorithm over a graph with unpredictable, data-dependent traversal order, running once on a small graph of 200 nodes.
